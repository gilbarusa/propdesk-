Backfill Tenant-Facing Lease PDFs

One-time copy of every countersigned lease's signed PDF from leases-signed (public) to documents/leases/<property>/<unit>/<lease_id>.pdf (private), then patches leases.tenant_pdf_bucket + leases.tenant_pdf_path. The SQL migration's retrofit UPDATE (re-run it after this finishes) then propagates the private path onto existing lease_document_assignments rows, and the bridge trigger re-mirrors to tenant_documents.

Preconditions
1. Migration 2026-04-18-tenant-facing-lease-pdf.sql has been applied (adds tenant_pdf_bucket / tenant_pdf_path columns, widens the partial unique index).
2. A private storage bucket named documents exists in Supabase Storage.
3. You are an admin — this tool talks to Supabase directly with the anon key under the existing RLS policies.

Step 3 — After backfill finishes

Paste this into the Supabase SQL editor to propagate the private file_url onto existing lease_document_assignments rows:

update lease_document_assignments a
   set file_url    = l.tenant_pdf_bucket || '/' || l.tenant_pdf_path,
       file_bucket = l.tenant_pdf_bucket,
       file_path   = l.tenant_pdf_path,
       source_type = 'upload',
       updated_at  = now()
  from leases l
 where a.lease_id       = l.id
   and a.auto_generated = true
   and a.document_type  = 'lease'
   and a.source_type    = 'lease_link'
   and a.file_url       is null
   and l.tenant_pdf_bucket is not null
   and l.tenant_pdf_path   is not null;
